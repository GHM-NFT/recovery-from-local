# GHM Column Source Map (why some columns stay blank)

This map explains where values should come from in the current scripts.

## Slug + token_id
- `slug` is generated by your autos script, and only appends `-token_id` when duplicate base slugs collide.
- It is **not guaranteed** to include token_id on every row.
- `media_filename` is the safer per-token storage filename when the token suffix updater is run.

## Green (auto/derived) columns you listed

These will stay blank unless their specific producer runs and required inputs exist:

- `alt_text_en`
  - Filled by autos function `fillAltTextEn_` in your export script (not by Single Kit).
- `status_json`, `missing_fields`
  - Filled by JSON gate function (`fillJsonGateAndWarnings_`) in export autos script.
- `category_id`, `subcategory_id`
  - Filled only if taxonomy map sheet exists + lookup keys exist.
- `caption_long_en`, `research_notes`, `sources_bibliography`, `sot_link`
  - Pulled from `GHM_SoT` only when slugs match.
- `meta_description_auto`
  - Filled only by `fillSeoAutos_` in export autos script.

## Blue/admin-once columns you listed

The debugged Single Kit now fills these from `Control` (if blank):

- `royalty_bps` (from `royalty_bps_default`)
- `fee_recipient` / `royalty_recipient` (from `fee_recipient_default`)
- `reveal_mode` (from `reveal_mode_default`)
- `reveal_date` (from `reveal_date_default`)
- `freeze_policy` (from `freeze_policy_default`)
- `freeze_date` (from `freeze_date_default`)
- `primary_mint_platform` (from `primary_mint_platform_default`)
- `est_gas_mint` (from `est_gas_mint_default`)
- `est_gas_batch` (from `est_gas_batch_default`)

Also:
- `token_range` now backfills from `token_id` when blank.
- `collection_path` now backfills as `04_COLLECTIONS/<Pantheon|Series>/<Character|Title/Name|name_final>` when blank.

## About repeated `collection_path`
Repeated folder path can be intentional if multiple NFTs belong to one character folder.
Uniqueness should come from filenames (`media_filename`, `json_filename`) rather than folder path.

Taxonomy_Mapping needs headers: Series, Character, Character_Variant, category_id (and optionally subcategory_id, taxonomy_tags).
